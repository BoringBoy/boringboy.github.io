linux 中sed、awk、grep及正则表达式的用法

# sed-流编辑

sed（Stream Editor）是一个流式文本编辑器，用于处理和转换文本数据。它按行读取输入，可以对文本进行替换、删除、插入和其他操作，并将结果输出到标准输出。

# 语法

sed [option] [files]

# 基本使用

  1. 执行多个命令, -e 选项
    sed -e "s/hello/hi/;s/dog/cat/" ./file.txt
    1
  2. 执行命令文件，-f 选项
    sed -f script.sed ./data.txt
    1
    执行"cat script.sed"，可看到文件中命令如下所示：
    s/bb/aa/
s/red/green/
....
    1
    2
    3
  3. 多个命令还可以用下面组合的方式
    sed '3{
s/local/global/
s/hello/hi/
....
}' ./data.txt
    1
    2
    3
    4
    5

  • 替换 (s)
    （1）字符替换

     sed "s/cat/dog/" ./a.txt
    1

    （2）管线形式字符替换

     echo "helloworld" | sed "s/hello/welcome/"
    1

    替换选项

    1. g :替换所有匹配文本
    2. 数字 ：替换第几处匹配
    3. w file ：将替换结果写入到文件

    示例：将data.txt文件中的匹配结果写入到b.txt中

    sed "s/dog/cat/w b.txt" ./data.txt
    1
    • 指定行替换
    sed "2,$s/dog/cat/" ./data.txt
    1
    • 指定文本行替换
    sed "/user/s/aaa/bbb/" ./data.txt
    1

  • 删除行 (d)

    sed '3d' ./data.txt         #删除第三行
sed '3,10d' ./data.txt      #删除3-10区间的行
sed '/jack/d' ./data.txt    #删除包含“jack”的行
    1
    2
    3
    4

  • 插入行 (i) ：指定行前增加新行

    sed "3i\hello world" ./data.txt     # 将特定行插入到data.txt的第三行前
    1

  • 附加行 (a) : 指定行后增加新行

  • 修改行 (c) : 修改整行内容

    sed "3c\this is new line" ./data.txt
sed "/home/c\this is new line" ./data.txt
    1
    2

  • 转换命令 (y): 字符一一映射的替换

  • 打印行 (p)

    #打印替换后的行
sed -n "s/hello/world/p" ./data.txt

# gawk

awk是一种用于处理和分析文本数据的编程语言。它以行为单位读取输入，并允许指定如何处理每一行。awk使用一种类似于C语言的语法，并提供了许多内置函数和特殊变量，用于访问和操作输入数据。可以使用awk来提取和打印特定列，基于条件进行过滤和计算，以及执行其他高级文本处理任务。

# 语法

gawk options program file
1

# 基本使用

  1. 可选变量

    (1) -F :指定分隔符，默认分隔符为空格

    #示例：使用冒号做分隔符
gawk -F: '{print $1,$2}' /etc/passwd
    1
    2

    (2) -f :用于从文件中读取程序

    #示例:指定myscript.gawk 做程序文件
gawk -F: -f ./myscript.gawk /etc/passwd
    1
    2

  2. 数据字段变量。 gawk 默认以空格分割每一行的字段，其中
    (1) $0 : 整个文本行
    (2) $1 : 文本行第一个字段
    (3) $n :文本行第n个字段
    示例：

    #假如存在以下一行文本
    1 root      20   0  193724   6828   4180 S   0.0  0.4   0:03.76 systemd
        
    #使用gwak输出root,193724
    gawk '{print $2,$5}'

gawk 'BEGIN{print "now start"}{print $0}END{print "end!"}' /etc/passwd

# grep

grep是一种强大的文本搜索工具，用于在文件中查找特定的模式。它可以接受一个或多个正则表达式作为搜索模式，并在文件中寻找与之匹配的行。

# 语法

grep [选项] pattern [文件名]

# 基本使用

  1. 常用选项
常用选项：
　　-E ：开启扩展（Extend）的正则表达式。
　　-i ：忽略大小写（ignore case）。
　　-v ：反过来（invert），只打印没有匹配的，而匹配的反而不打印。
　　-n ：显示行号
　　-w ：被匹配的文本只能是单词，而不能是单词中的某一部分，如文本中有liker，而我搜寻的只是like，就可以使用-w选项来避免匹配liker
　　-c ：显示总共有多少行被匹配到了，而不是显示被匹配到的内容，注意如果同时使用-cv选项是显示有多少行没有被匹配到。
　　-o ：只显示被模式匹配到的字符串。
　　--color :将匹配到的内容以颜色高亮显示。
　　-A  n：显示匹配到的字符串所在的行及其后n行，after
　　-B  n：显示匹配到的字符串所在的行及其前n行，before
　　-C  n：显示匹配到的字符串所在的行及其前后各n行，context

  1. 示例
#单个文件中查找关键词
grep 'Class 1' Students.txt

#从多个文件中查找关键词
grep "omc" /etc/passwd /etc/shadow

#查找多个字符串
grep -e 'Class 1' -e Todd Students.txt

#可以将-r开关与grep一起使用，以递归方式搜索目录及其子目录中的所有文件
grep -r pattern /directory/to/search

#查找string1 或 string2
grep -E 'string1|string2' filename

# B   查询匹配内容的一行之外，前n行的显示

# 正则表达式

正则表达式是一种模式匹配语言，用于在文本中搜索和匹配特定的字符模式。

# 基本正则

  1. 行首(^)
  2. 行尾($)
  3. 点号(.) :匹配换行符外的任意单个字符
  4. 字符组([]) :匹配希望出现在方括号中的字符
    sed "s/[aA]rm/ban/" ./data.txt
    1
  5. 排除型字符组,即匹配字符组外的字符，表现形式为字符组前面加 ^ 字符。示例如下：
    sed -n '/[^abc]at/p' ./data.txt
    1
  6. 区间:用来匹配某个区间的字符，表现形式通常为区间首末字符中添加 - 字符，示例如下：
    sed -n '/[0-6]ye/p' ./data.txt
    1
  7. 星号(*):
    字符后面放置星号表示该字符必须在匹配模式的文本中出现0次或多次，示例如下：
    sed -n '/u*nion/p' ./data.txt
#可以匹配到
union aaab
uunion abaf
anion abga

#不能匹配到
uonion aiitap

该部分适用于gwak,但不支持sed

    '/a?bp/'

  3. 管道符号( | ) ：匹配其中一个模式
