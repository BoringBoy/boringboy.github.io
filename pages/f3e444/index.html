# centos/redhat包管理

本篇文章整理记录了centos/redhat系统中包管理的相关概念和常用命令，并给出离线安装包时缺少依赖的几种解决办法。

# 基本知识

# 1. yum 与 rpm 区别？

yum基于RPM包管理，能够从指定的服务器自动下载RPM包并且安装，可以自动处理依赖性关系，并且一次安装所有依赖的软件包，无须繁琐地一次次下载、安装。

# 2.".repo"文件是什么？

  • .repo文件是什么

repo文件是Fedora中yum源（软件仓库）的配置文件，通常一个repo文件定义了一个或者多个软件仓库的细节内容，例如我们将从哪里下载需要安装或者升级的软件包，repo文件中的设置内容将被yum读取和应用！

  • .repo文件通常所在位置?

/etc/yum.repos.d/

# 常用命令

# yum 命令

# 安装软件包
sudo yum install [package]

# 删除软件包及其依赖
sudo yum remove [package]

# 升级软件包
sudo yum update [package]

# 查看软件包信息
sudo yum info [package]

# 清理本地缓存
sudo yum clean all 
# 清理插件缓存
sudo yum clean plugins

# 构建缓存
sudo yum makecache

# 查找软件包
sudo yum search [package]

# 查看包依赖信息
sudo yum deplist [package]

# 显示可用仓库
sudo yum repolist enabled

# 显示已安装包
yum list installed

# 显示可更新包
yum list updates

# rpm 命令(全称：RedHat Package Manager)

  • 安装rpm 包
# -i:安装，-v:显示详细信息,-h:显示安装进度
rpm -ivh [package]

# 忽略报错强制安装
rpm --force -ivh [package]

# 不检测依赖安装
rpm --nodeps -ivh [package]

# -U（大写）选项的含义是：如果该软件没安装过则直接安装；若没安装则升级至最新版本。
rpm -Uvh [pacakge]

# 解决离线安装缺少依赖的问题

# 方法1：使用downloadonly插件

说明：该种方法仅会将主软件包和基于你现在的操作系统所缺少的依赖关系包一并下载。如果指定的rpm包已经在你的os上安装结束，那么下面命令就得不到想要的结果

  1. 安装插件
sudo yum install yum-plugin-downloadonly
1
  1. 安装好之后执行下载命令
# 执行下面命令
sudo yum install --downloadonly <package-name>

# 也可通过 -–downloaddir 指定下载目录
sudo yum install --downloadonly --downloaddir=/temp/ <package-name>

# 方法2：使用Yumdownloader软件包

  1. 安装Yumdownloader工具
sudo yum install yum-utils
1
  1. 开始下载软件包
# -–destdir：指定 rpm 包下载目录（不指定时，默认为当前目录）
# –-resolve：下载依赖的 rpm 包。
#使用 --destdir 参数，将软件下载到一个特定的目录下
#示例：
sudo yumdownloader --resolve --destdir=/root/mypackages/ vsftpd

# 方法3：使用repotrack（推荐）

  1. 首先安装repotrack工具
sudo yum install yum-utils
1
  1. 开始下载rpm包
#下载rpm包(默认当前目录)
# 示例：repotrack vsftpd
repotrack [package_name]

#下载rpm包(指定下载目录，使用-p)
#示例--下载离线包至当前目录下的temp文件夹:

repotrack -p ./temp [package_name]
